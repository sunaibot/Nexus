# 插件构建器快速入门

## 🎯 5分钟上手

### 第一步：理解概念

```
零件 (Part) = 乐高积木块
插件 (Plugin) = 用积木搭成的模型
```

**零件类型：**
- 🔘 按钮、📝 文本、🃏 卡片 - 基础UI
- 📊 图表、📋 列表 - 数据展示  
- 📥 输入框、🎚️ 滑块 - 交互组件
- 📐 布局容器 - 排列其他组件

### 第二步：打开零件工坊

1. 访问管理后台：`http://localhost:5174`
2. 进入「插件中心」→「零件工坊」
3. 看到系统预设的零件库

### 第三步：创建第一个零件

**目标：创建一个带图标的按钮**

1. 点击「新建零件」
2. 填写基本信息：
   - 名称：图标按钮
   - 分类：basic
   - 图标：🔘

3. 配置视觉样式：
```css
/* 基础样式 */
padding: 10px 20px
background-color: #3b82f6
color: #ffffff
border-radius: 8px
```

4. 添加悬停效果：
```css
/* 悬停状态 */
background-color: #2563eb
transform: translateY(-2px)
box-shadow: 0 4px 12px rgba(59,130,246,0.4)
```

5. 定义可配置属性：
   - 文字（字符串）
   - 图标（选择）
   - 颜色（颜色选择器）

6. 保存零件

### 第四步：使用插件构建器

1. 进入「插件构建器」标签
2. 点击「新建插件」
3. 设置画布大小（建议 800x600）
4. 从左侧面板拖拽零件到画布

**搭建一个简单的名片插件：**

```
┌─────────────────────────┐
│  🃏 卡片容器              │
│  ┌───────────────────┐  │
│  │ 👤 头像图片        │  │
│  │ 📝 姓名（大标题）   │  │
│  │ 📝 职位（小文本）   │  │
│  │ 🔘 联系按钮        │  │
│  └───────────────────┘  │
└─────────────────────────┘
```

### 第五步：添加数据

**让名片显示真实数据：**

1. 点击「数据源」面板
2. 添加API数据源：
   - 名称：用户信息
   - 接口：/api/users/profile
   - 方法：GET

3. 选中姓名文本组件
4. 在属性面板绑定数据：
   - 内容：`{{用户信息.name}}`

5. 选中职位文本组件
   - 内容：`{{用户信息.title}}`

### 第六步：添加交互

**让按钮可点击：**

1. 选中按钮组件
2. 点击「事件」标签
3. 添加点击事件：
   - 动作类型：打开链接
   - 链接：`mailto:{{用户信息.email}}`

### 第七步：预览和保存

1. 点击「预览」按钮查看效果
2. 点击「保存」存储插件
3. 设置插件名称和图标
4. 选择发布范围（公开/私有）

## 📚 常用操作速查

### 画布操作

| 操作 | 快捷键/方式 |
|------|------------|
| 移动组件 | 拖拽 |
| 多选 | Ctrl + 点击 或 框选 |
| 调整大小 | 拖拽边缘手柄 |
| 删除组件 | Delete 键 |
| 复制组件 | Ctrl + C / V |
| 撤销 | Ctrl + Z |
| 放大/缩小 | Ctrl + 滚轮 |

### 数据绑定语法

```
{{数据源.字段名}}          - 简单绑定
{{用户.name}}             - 嵌套字段
{{列表.length}}           - 数组长度
{{价格}} * {{数量}}        - 计算
{{状态}} === "active"      - 条件
```

### 组件对齐

选中多个组件后：
- 左对齐 / 右对齐 / 水平居中
- 顶部对齐 / 底部对齐 / 垂直居中
- 水平分布 / 垂直分布

## 🎨 实战：复刻名言插件

### 分析需求

原名言插件功能：
- 显示一句名言
- 显示作者
- 点击换一条

### 选择零件

需要：
1. 🃏 卡片容器 - 作为背景
2. 📝 文本（大）- 显示名言
3. 📝 文本（小）- 显示作者
4. 🔘 按钮 - 换一条

### 搭建步骤

**1. 创建卡片**
- 拖拽「卡片容器」到画布中央
- 设置尺寸：400x200
- 设置背景：半透明深色

**2. 添加名言文本**
- 在卡片内添加「标题文本」
- 设置：居中、斜体、白色
- 绑定数据：`{{名言.content}}`

**3. 添加作者**
- 在名言下方添加「文本」
- 设置：右对齐、小字号、灰色
- 绑定数据：`—— {{名言.author}}`

**4. 添加刷新按钮**
- 在卡片右下角添加「幽灵按钮」
- 设置文字："换一条"
- 添加点击事件：刷新数据源

**5. 配置数据源**
- 添加API数据源
- 接口：/api/quotes/random
- 绑定到名言变量

**6. 完成**
- 预览测试
- 保存为「每日名言」插件

## 💡 技巧提示

### 1. 快速复制样式

选中组件 → 右键「复制样式」→ 选中其他组件 → 「粘贴样式」

### 2. 使用网格对齐

开启「显示网格」+「吸附网格」，组件会自动对齐

### 3. 组件组合

选中多个组件 → 右键「组合」，可以一起移动和调整

### 4. 响应式设计

使用百分比宽度：
- 宽度：100%（填满容器）
- 最大宽度：400px（限制最大尺寸）

### 5. 条件显示

设置组件的「可见性」属性：
```
{{用户.role}} === "admin"
```

只在用户是管理员时显示

## ❓ 常见问题

**Q: 数据绑定不生效？**
A: 检查：
1. 数据源ID是否正确
2. 字段名是否匹配
3. API是否返回数据

**Q: 组件重叠了怎么办？**
A: 使用：
1. 调整zIndex（层级）
2. 使用布局容器包裹
3. 开启网格对齐

**Q: 如何调试？**
A: 使用：
1. 预览模式查看效果
2. 浏览器F12查看网络请求
3. 检查数据源返回的数据结构

**Q: 插件能在前台显示吗？**
A: 保存后：
1. 启用插件
2. 设置可见性为「公开」
3. 前台会自动加载
